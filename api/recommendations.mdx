## Overview

The recommendation endpoints use OpenAI's GPT-3.5-turbo model to generate personalized, actionable recommendations for farmers based on their current environmental conditions and optimal ranges for their chosen crop.

All recommendation endpoints accept the same request schema and return AI-generated text recommendations.

## Common Request Schema

All recommendation endpoints use the `FarmData` model:

<ParamField body="min" type="float" required>
  Minimum recommended value for the environmental factor
</ParamField>

<ParamField body="max" type="float" required>
  Maximum recommended value for the environmental factor
</ParamField>

<ParamField body="value" type="float" required>
  Current measured value on the farm
</ParamField>

<ParamField body="label" type="string" required>
  Crop type (e.g., "rice", "maize", "cotton")
</ParamField>

## Temperature Recommendations

<CodeGroup>
```bash POST /api/recommedations/temperature

  -H "Content-Type: application/json" \

    "max": 30.0,

    "label": "rice"

<ResponseField name="response" type="string">

**Example Response:**
```json

To improve conditions, consider using shade nets to reduce direct sunlight, increase irrigation frequency to cool the soil, and plant heat-tolerant rice varieties. Mulching can also help regulate soil temperature effectively."

## Implementation Details

The recommendation engine (`recommendation_engine.py:9`) uses the following logic:

1. Accepts factor name, min/max range, current value, unit, and crop type
2. Sends a request to OpenAI's GPT-3.5-turbo model with a system prompt optimized for rural farmers
3. Generates a dynamic, 55-word recommendation block
4. Returns simplified, actionable strategies specific to the crop
5. All numeric values in responses are approximated to 2 decimal places

<Note>
**API Key Required:** These endpoints require an OpenAI API key configured in the `.env` file as `OPENAI_API_KEY`. Without this key, the recommendation endpoints will fail.
</Note>

## Best Practices

<AccordionGroup>
  <Accordion title="Use Accurate Ranges">
    Ensure that the `min` and `max` values you provide match the optimal ranges for your specific crop. Inaccurate ranges will result in misleading recommendations.
  </Accordion>
  
  <Accordion title="Current Value Measurement">
    The `value` parameter should reflect actual measurements from your farm using calibrated sensors or testing equipment.
  </Accordion>
  
  <Accordion title="Crop Label Format">
    Crop labels must be lowercase and match the supported crop list exactly (e.g., "kidneybeans" not "kidney beans" or "Kidneybeans").
  </Accordion>
  
  <Accordion title="Interpretation">
    Recommendations are AI-generated and should be interpreted as suggestions. Always consult with local agricultural experts before making significant changes to farming practices.
  </Accordion>
</AccordionGroup>
